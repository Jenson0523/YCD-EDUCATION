# 架构设计

云辰盾采用四层架构：基础支撑层、统一数据中台、核心业务应用层、多终端应用层。

## 基础支撑层

负责云服务器、安全加密、数据备份、人脸识别、定位、电子签章、电子票据、音视频和消息推送等基础能力。

## 统一数据中台

数据中台是跨模块数据聚合和指标计算的唯一入口。业务模块只负责领域数据写入，中台负责画像、排名、预警、统计、绩效、驾驶舱。

关键原则：

- 学生 ID 是成长档案聚合主键。
- 跨模块统计走中台接口。
- 指标计算统一沉淀在中台，前端只展示结果。

## 核心业务应用层

固定 10 大板块，每个板块拥有独立包、路由、表前缀、README 和接口命名空间。

## 多终端应用层

- PC 管理后台：校长、股东、教务、人事、财务、招生、行政、班主任、心理安保校医。
- 微信小程序：教师、班主任、家长、学生。

## 关键横向能力

- RBAC 角色权限和数据权限。
- 全量操作日志。
- 统一审批流引擎。
- 消息通知解耦。
- 敏感数据加密和日志脱敏。
