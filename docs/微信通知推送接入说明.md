# 微信订阅消息（"微信通知"）接入说明

> 现状：**站内通知已全部生效**——首页弹窗、未读角标实时刷新、消息中心，本地即可测试。
> "微信通知"（用户在微信聊天列表收到服务通知、点击进入小程序）属于微信「订阅消息」，
> 受微信平台规则限制，**必须在发布版小程序 + 配置模板 + 用户授权**后才能用，本地开发无法测试。
> 系统已预留 `WxNotifyService`，配置好后自动生效。

## 一、为什么本地测不了
微信订阅消息要求：
1. 已发布（或体验版）的小程序，拥有 **AppID + AppSecret**
2. 在微信公众平台「功能 → 订阅消息」申请**消息模板**，拿到模板ID
3. 用户在小程序里点击按钮时 `wx.requestSubscribeMessage` **逐次授权**（微信强制：一次授权发一次）
4. 后端用用户的 **openid** 调微信API下发

## 二、接入步骤（上线后）

### 1. 绑定用户 openid（前置）
当前登录是「用户名+密码」，微信推送需要 openid。需在小程序首次进入时：
```js
// 小程序登录后调用
uni.login({ success: (r) => {
  request({ url:'/auth/wx-bind', method:'POST', data:{ code: r.code } })
}})
```
后端用 code 换 openid（`https://api.weixin.qq.com/sns/jscode2session`）存入 sys_user.openid。
> 该接口与 sys_user.openid 字段需新增——上线前告诉我，我补上（约30分钟）。

### 2. 配置（application-prod.yml 或环境变量）
```yaml
wx:
  appid: 你的小程序AppID
  secret: 你的小程序AppSecret
  tmpl-approve: 请假审批结果模板ID
```

### 3. 申请订阅消息模板
公众平台 → 订阅消息 → 选用/申请模板，例如"请假审批结果通知"，字段如：
- 学生姓名 `thing1`
- 审批结果 `phrase2`
- 审批时间 `time3`
拿到模板ID填上面 `tmpl-approve`。

### 4. 用户授权（小程序端）
在"提交请假"成功后引导授权：
```js
wx.requestSubscribeMessage({ tmplIds: ['模板ID'], success(){} })
```

### 5. 后端下发
审批通过时（`LeaveApplicationController.approve`）调用：
```java
wxNotifyService.send(parentOpenid, tmplApprove, data, "pages/leave/detail?id="+id);
```
`WxNotifyService` 已实现 access_token 缓存与下发，配置齐全即工作；未配置则静默跳过。

## 三、点击进入小程序
订阅消息的 `page` 字段已设为 `pages/leave/detail?id=xxx`，用户在微信点通知即可直达对应页面。

## 四、替代方案（若不想接订阅消息）
- **公众号模板消息**：需关联认证服务号，用户关注公众号后可推送（不限次，但需引导关注）。
- 当前**站内弹窗 + 未读提示**已能满足"打开小程序即见通知"，多数校园场景够用。
